# claude_system_prompts_full.md ができるまで

2026/07/04

対象ページ: https://platform.claude.com/docs/en/release-notes/system-prompts

## 最初の壁: curlしても中身が空(に見えた)

`curl <url> > page.html` しても、HTMLから`<script>`タグを除いて中身を見ると
`Loading...` ばかりで本文がない ―― ように見えた。

ネタバレ: 本文はその1回のレスポンスの中に、`<script>`タグ内のJSON文字列
として最初から丸ごと埋まっていた。追加の通信は一切発生しない。`Loading...`
はReactの「Suspense」というフォールバック表示で、ブラウザはすぐ近くの
`<script>`に埋まってるJSONを読み取ってそれを一瞬で本物の中身に差し替える
だけ。「JSが後から取りに行く」んじゃなくて、「最初から手元にあるものを
JSが読み解いて表示に反映する」というのが正確なところ。だから普通に
`<script>`タグを除去してテキストを取り出すと、本文ごと消えてしまってた。

## 次の壁: WebFetchは「要約」してしまう

Claude Code の `WebFetch` ツールを使えばレンダリング後の内容が取れるが、
これは裏で小さいモデルが「要約・言い換え」して返す仕組み。今回は全システム
プロンプトの逐語的な全文が欲しかったので、ページが長いとモデルが途中で
勝手に切り詰めたり、表現を変えてしまったりして、分析用としては使えなかった。

## ブレイクスルー: 生HTMLの中にRSCストリームとして全文が埋まっていた

よく見ると、最初にcurlした生HTMLの中に `self.__next_f.push([1,"..."])` という
形でReact Server Components(RSC)のストリームデータがそのままJSON文字列として
埋め込まれていた。つまり全文は最初からローカルにあった。あとはこれを正しく
組み立て直すだけ、のはずが、実際には3つのバグにハマった:

1. チャンクの区切りが必ずしも改行(`\n`)と一致しない — 単純に`\n`で分割すると
   レコードが2つ結合されて後半が消える
2. 一部のデータは通常のJSONではなく `id:Txxx,<生バイト列>` という特殊な
   テキストチャンク形式で入っていて、これを普通にJSONパースしようとすると
   無言で失敗し、それを参照している箇所の本文がごっそり消える
3. 参照記法が `$Lxx`(要素への参照)と `$xx`(テキストチャンクへの参照、Lなし)
   の2種類あり、片方しか処理していないと一部モデルの本文が空になる

これらを潰して、公式ページの一覧と件数が完全一致(16モデル・28件の日付
エントリ)するところまで検証して完成させた。

## この技術は他のMintlify系ドキュメントサイトにも使い回せるはず

Next.jsのRSCストリームからSPAの全文を復元するというやり方自体は、この
ページ固有のものではなく、Mintlify(や同種のNext.js製ドキュメントサイト)
全般に効くはず。同じ壁(curlしても空、WebFetchだと要約されて崩れる)に
ぶつかったら、まず `scripts/extract_nextjs_rsc.py` を試すとよい。

```
curl <mintlify系docsのURL> > page.html
python3 scripts/extract_nextjs_rsc.py page.html -o output.md
```

ハマりどころ(上記3点)はスクリプトのdocstringにも書いてある。ページの
レイアウトが今回のようなアコーディオン形式のチェンジログでない場合は、
見出し/日付タイトルの判定ロジック(`heading_level` / `title` まわり)を
調整する必要があるかもしれない。

なお、生HTML(`page.html`)は検証後に削除済み。
再度必要になったら上のcurlコマンドで取り直せる(ページは新モデルが出る
たびに更新されるので、都度最新を取り直すのが前提)。
